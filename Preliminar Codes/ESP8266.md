# STM32F446RE & ESP8266MOD UART Communication Module

This repository contains the implementation of an embedded communication system utilizing the **UART** protocol to interconnect an **STM32F446RE** development board (ARM Cortex-M4) and an **ESP8266 (ESP-12E / NodeMCU)** Wi-Fi module.

The primary objective is to establish a reliable, bidirectional communication channel, enabling seamless command and data exchange between both microcontrollers.

---

## 🔌 Hardware Connection (Pinout)

The connection requires crossing the transmission (`TX`) and reception (`RX`) lines, alongside ensuring a stable electrical reference.

Since pins **D5** and **D6** on the ESP8266 are not part of the native hardware UART peripheral (`UART0`), the ESP8266 firmware emulates a serial port via software using the **SoftwareSerial** library.

### Connection Table

| Device 1: STM32F446RE | Device 2: ESP8266 (NodeMCU) | Function | Technical Description |
| :--- | :--- | :--- | :--- |
| **PA11** (e.g., USART6_RX) | **D5** (GPIO14 - SCLK) | **STM_RX ← ESP_TX** | ESP8266 transmission line emulated via SoftwareSerial. |
| **PA12** (e.g., USART6_TX) | **D6** (GPIO12 - MISO) | **STM_TX → ESP_RX** | ESP8266 reception line emulated via SoftwareSerial. |
| **GND** | **GND** | **Common Ground** | **Crucial:** Unifies the voltage reference for both logic signals. |
| **USB (PC)** | **USB (PC)** | **Power** | Both boards are independently powered through the computer's USB bus. |

> ⚠️ **Hardware Note:** Even though both boards are independently powered by the computer, **connecting the GND pins together is strictly mandatory**. Without this common ground, the UART data signals will lack a proper reference, causing floating states, corrupted data, or a complete failure to recognize the 3.3V logic levels.

---

## 🛠️ Peripheral Configuration

* **ESP8266:** Configured using the `SoftwareSerial` library, assigning `D5` as TX and `D6` as RX at a standard baud rate (e.g., `9600` or `115200` bps, depending on stability requirements).
* **STM32F446RE:** Configured via **STM32CubeIDE** by enabling the corresponding USART peripheral in Asynchronous Mode, matching the baud rate, and utilizing either interrupts (`HAL_UART_Receive_IT`) or DMA for efficient buffer management.
